Spring AI 2.0 Demo Spring Boot 4 · Spring AI 2.0 · Anthropic (claude-sonnet-4-5)

스터디 문서 spring-ai-study.md의 예제를 실제로 실행합니다. 각 섹션은 무엇을 보여주는지와 핵심 코드를 함께 표시하고, 오른쪽 태그가 문서의 해당 장(§)입니다. 문서 §7(MCP)은 MCP 클라이언트가 필요해 브라우저 시연이 없습니다.

1. 기본 채팅 ChatClient문서 §1

가장 기본. LLM에 메시지를 보내고 텍스트 응답을 받습니다.

chatClient.prompt().user(msg).call().content()

2. 스트리밍 .stream()문서 §1

응답을 한 번에 받지 않고 토큰이 오는 대로 흘려보냅니다(타이핑 효과). 반환은 Flux → SSE.

chatClient.prompt().user(msg).stream().content()  // Flux<String>

3. 시스템 프롬프트 .system()문서 §1

페르소나(말투·역할)를 지정합니다. 여기선 "해적처럼 말하는" 챗봇.

.system("너는 해적처럼 말한다").user(msg)

4. Structured Output .entity()문서 §2

텍스트가 아니라 타입 객체(JSON)로 받습니다. LLM을 타입 안전한 함수처럼 사용.

.call().entity(Movie::class.java)   // {title, year, director}

5. 대화 메모리 ChatMemory문서 §3

LLM은 원래 매 호출을 따로 처리해 이전 대화를 기억하지 못합니다. 같은 대화 ID로 보내면 이전 메시지를 함께 넣어줘서 맥락을 기억합니다.

확인 방법: ① 버튼 → 전송 (이름 알려주기) → ② 버튼 → 전송 (같은 ID로 이름 되묻기). 기억하면 성공입니다.

영속화: 대화는 메모리가 아니라 DB(H2 파일)에 저장됩니다. 앱을 껐다 켠 뒤 ②를 눌러도 이름을 기억합니다. 운영에서는 같은 코드로 저장소만 Postgres·Redis로 바꿉니다.

.advisors { it.param(CONVERSATION_ID, id) }   // 같은 ID = 같은 대화

MessageWindowChatMemory.builder()      // 정책: 최근 N개만 유지
    .chatMemoryRepository(repo)        // 저장소: JDBC (교체 가능)
    .build()

6. Tool Calling @Tool문서 §4

LLM이 우리 함수(getWeather)를 필요할 때 직접 호출합니다. "서울 날씨"를 물으면 도구가 실행됩니다.

.tools(WeatherTools())   // @Tool fun getWeather(city)

7. RAG QuestionAnswerAdvisor문서 §5

사내 문서(환불 정책·영업시간)를 검색해 근거로 답합니다. 시드된 문서: 환불 7일 이내, 영업 평일 09~18시.

.advisors(QuestionAnswerAdvisor.builder(vectorStore).build())

8. ToolSearch ToolSearchToolCallingAdvisor문서 §6

도구가 여러 개(날씨·덧셈·주문조회·시간)일 때, 질문에 맞는 것만 골라 LLM에 노출합니다.

전송하면 availableTools에 후보 도구 전체가, reply에 그중 실제로 쓰인 결과가 표시됩니다.

.advisors(toolSearchAdvisor).tools(ManyTools())

9. 토큰 사용량 관측성 · Usage문서 §9

LLM은 토큰 단위로 과금됩니다. 그래서 호출이 토큰을 얼마나 썼는지 추적하는 게 중요합니다. 응답의 content() 대신 chatResponse()로 받으면 메타데이터에서 토큰 수를 읽을 수 있습니다.

response.metadata.usage.totalTokens

전송하면 응답과 함께 promptTokens(입력), completionTokens(출력), totalTokens(합계)가 표시됩니다. 서버 콘솔에는 모델에 나간 프롬프트 전문도 찍힙니다(log-prompt — RAG·메모리가 프롬프트를 어떻게 바꿨는지 여기서 보입니다).

호출 단위 조회와 별개로, Micrometer는 모든 호출의 토큰을 지표로 누적합니다(임베딩 포함). actuator 엔드포인트를 그대로 조회해 봅니다.

GET /actuator/metrics/gen_ai.client.token.usage

10. 커스텀 Advisor BaseAdvisor문서 §8

앞의 RAG·메모리·ToolSearch가 모두 Advisor였습니다. 직접 만들 수도 있습니다. BaseAdvisor를 구현하고 before(요청)/after(응답)만 채우면 됩니다 — HandlerInterceptor와 같은 자리입니다.

여기서는 카드번호를 마스킹해서 LLM에 보내는 가드레일입니다. 아래 모델이 실제로 받은 문장을 보면 번호가 가려진 채 나간 것을 확인할 수 있습니다.

class PiiMaskingAdvisor : BaseAdvisor {
    override fun before(req, chain) = // 카드번호 마스킹 후 요청 교체
    override fun after(res, chain) = res
}
