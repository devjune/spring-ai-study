Spring AI 2.0 Demo Spring Boot 4 · Spring AI 2.0 · Anthropic (claude-sonnet-4-5)

스터디 문서 spring-ai-study.md의 예제를 실제로 실행합니다. 각 섹션은 무엇을 보여주는지와 핵심 코드를 함께 표시합니다.

1. 기본 채팅 ChatClient

가장 기본. LLM에 메시지를 보내고 텍스트 응답을 받습니다.

chatClient.prompt().user(msg).call().content()

2. 스트리밍 .stream()

응답을 한 번에 받지 않고 토큰이 오는 대로 흘려보냅니다(타이핑 효과). 반환은 Flux → SSE.

chatClient.prompt().user(msg).stream().content()  // Flux<String>

3. 시스템 프롬프트 .system()

페르소나(말투·역할)를 지정합니다. 여기선 "해적처럼 말하는" 챗봇.

.system("너는 해적처럼 말한다").user(msg)

4. Structured Output .entity()

텍스트가 아니라 타입 객체(JSON)로 받습니다. LLM을 타입 안전한 함수처럼 사용.

.call().entity(Movie::class.java)   // {title, year, director}

5. 대화 메모리 ChatMemory

같은 대화 ID면 이전 맥락을 기억합니다. 먼저 "내 이름은 준이야" 전송 → 이후 "내 이름 뭐야?"로 물어보세요.

.advisors { it.param(CONVERSATION_ID, id) }   // 2.0: 호출 시점 지정

6. Tool Calling @Tool

LLM이 우리 함수(getWeather)를 필요할 때 직접 호출합니다. "서울 날씨"를 물으면 도구가 실행됩니다.

.tools(WeatherTools())   // @Tool fun getWeather(city)

7. RAG QuestionAnswerAdvisor

사내 문서(환불 정책·영업시간)를 검색해 근거로 답합니다. 시드된 문서: 환불 7일 이내, 영업 평일 09~18시.

.advisors(QuestionAnswerAdvisor.builder(vectorStore).build())

8. ToolSearch ToolSearchToolCallingAdvisor

도구가 여러 개(날씨·덧셈·주문조회·시간)일 때, 질문에 맞는 것만 골라 LLM에 노출합니다.

.advisors(toolSearchAdvisor).tools(ManyTools())

9. 관측성 Usage

응답 메타데이터에서 토큰 사용량을 읽습니다(비용·모니터링).

response.metadata.usage.totalTokens

10. MCP @McpTool

이 앱 자체가 MCP 서버입니다. 아래 도구를 MCP 프로토콜로 노출해, 다른 에이전트(MCP 클라이언트)가 표준 규격으로 호출할 수 있습니다. (이 화면에서 직접 호출하는 대신, MCP 클라이언트로 접속해 확인)

@McpTool(name="add")  fun add(a, b) = a + b
@McpTool(name="echo") fun echo(message) = message
MCP 엔드포인트로 tools/list 하면 add, echo 가 노출됩니다.
